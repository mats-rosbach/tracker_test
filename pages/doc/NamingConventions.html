Naming conventions

Why?

We collect web behavior data from several places across the digital bank, and everything ends up in the same places. It is important that datasets look similar regardless of which team or application that has provided the data.

Common principles

Names should be in norwegian.

We have domain specific terms and language in the digital bank that is most familiar in Norwegian. Terms and language that is not easily translated to English. For instance, "Depsitumskonto" is easily confused with "Deposit Account" (at least according to Google Translate), which in turn has a very different meaning in English. In order to avoid these sorts of misunderstandings, we want identifiers such as names to be in Norwegian.

Avoid special characters

We're sending data to data warehouses, Humio and Adobe Analytics. Additionally, the data passes three layers of proxies on its way to those destinations. Identifiers containing special characters can be hard to use as input for analytics and could be translated or changed during transport. Spaces could for instance become urlencoded to %20. You should prefer using - or _ as separators between words. Occasionally | is fine

The rule of thumbs regarding special characters is:

  • No Norwegian characters (æ, ø, å)
  • No spaces in identifiers
  • No special characters such as /, ?, (backtick),:, ., &, %, #`
  • Minus (-) and underscore (_) is fine
  • Basically, try to use expressions which is valid towards the following regex a-zA-Z0-9\_-|

There are however some exceptions to the rule. For instance, when you track what a customer has searched on, we expect the actual user input to be tracked.

No sensitive or personal information

Personal information shall not be sent.

  • No card details
  • No account numbers
  • No hashes
  • No transaction or payment details
  • No id's that belong to the customer, except an anonymous id (Declared id) if the customer has consented

Based on the url, we'll try to detect if there are some personal identifiers (hashes or uuids) and replace those with the string "trfdata" ("transformed data"). If the customer for instance visits "/ostlandet/nettbank-privat/bevegelser/", the url will appear as "/ostlandet/nettbank-privat/bevegelser/trfdata" across the different destinations.

Page Name

Page names should describe a specific page or native view. It is usually formatted based on the url and is not something you regularly need to send manually.

  • It should be unique for a given page
  • It should have the same format on all pages.
  • It should not contain id's, keys, card numbers, account numbers or hashes. Those should be replaced with placeholders
  • It is independent of any query-parameters

Example

 
sb1|nb|ostlandet|nettbank-privat|betaling|registrer-betaling
  • Page names consists of sections of the urlpath separated by pipe (|).
  • Pages on sparebank1.no is prefixed with sb1 as first section.
  • Language is the second section.
  • The tail of the page name is the path of the url, with slashes being replaced by pipe.

Buttons

The button names consists of three components: section, product and action

  • section - Which section or area of the site does the button reside on
  • product - (optional) - Which product does the button affect?
  • action - Which action is performed by clicking the button?
  • buttonName - Name we will generate by concatenating section, product and action. ${section}_${product}_${action}.
  • buttonText - The text that is displayed to the user on the button.
  • buttonId - (optional) - Unique id for the button. Normally the combination of section, product and action will be enough to uniquely identify the button. However, there are some cases where an additional unique ID might be useful (open pages).
  • linkType - (optional) - The type of link that the button represents: LinkType.

Example: [Betaling] Faste overføringer

section betaling
product overforinger-egne-konti
action se-faste-overforinger
buttonName betaling_overforinger-egne-konti_se-faste-overforinger
buttonText Faste overføringer
linkType button
