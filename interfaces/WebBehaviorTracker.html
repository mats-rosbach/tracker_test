consentChanged

consentChanged: ((properties: ConsentChangedProps, options?: EventOptions) => void)

Type declaration

    • (properties: ConsentChangedProps, options?: EventOptions): void

    • When the customer performs changes to opt-ins, you should call consentChanged. This call is performed regardless of the users' opt-ins, in order for our consumers to be updated with the customers' choice. Values for how the consent has changed needs to be provided, this done by providing old consents and the new ones. A user that has previously opted out, but now opts in from i.e. functional would have previous set to false, and updated set to true Full example:

      const tracker = getTracker();

      tracker.consentChanged({
        consentName: "efaktura",
        previousFunctional: false, // The user was opted out
        previousStatistic: false, // The user was opted out
        previousMarketing: undefined, // (Alternatively do not pass the parameter). The user had neither opted in nor out.
        updatedFunctional: true, // The user is opting in
        updatedStatistic: true, // The user is opting in
        updatedMarketing: undefined // (Alternatively do not pass the parameter). The user is neither opting in nor out.
      });

      Parameters

      • properties: ConsentChangedProps
      • Optional options: EventOptions

      Returns void
